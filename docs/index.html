Why StoryAD-QA?

The StoryTeller paper studies training-free long-form audio description for movies. A strong AD system should not only describe the current frame; it should preserve narrative context, resolve recurring characters, and explain why events matter as the story unfolds.

Standard captioning metrics are useful but limited: lexical overlap does not directly measure whether a generated AD contains the information needed to understand the story. StoryAD-QA was built to evaluate that missing dimension. It asks multiple-choice questions about actions, events, and narrative context, and scores whether a model's generated description supports the correct answer.

This release contains QA annotations, answer keys, prompts, and evaluation code. It does not include movie videos or other media assets.

The paper describes construction over 2,729 generated questions. This repository contains a verified public release of 2,572 QA pairs. We provide combined files plus question-only and answer-key splits.

Evaluation Code

The release includes a lightweight evaluator for multiple-choice accuracy. During evaluation, the answering model should receive only generated AD text, the question, and answer choices. 

pip install -r evaluation/requirements.txt
python evaluation/evaluate_storyad_qa.py \
  --answers data/StoryAD-QA/movie_qa_v5_30s_gemini-3-flash-preview.csv \
  --predictions predictions/my_model_v5_30s.csv

StoryTeller

StoryTeller is a training-free framework for long-form audio description with an identity graph, grounded facts, and narrative memory. StoryAD-QA is released as the QA-based evaluation benchmark used to test whether AD outputs preserve story understanding. The benchmark complements lexical AD metrics by asking answerable questions about the generated narrative.
